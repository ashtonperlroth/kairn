# v1.13.0 Design Doc — Environment Bootstrapping

> Before an agent starts reasoning, give it a snapshot of what it's working with. Eliminates 2-5 wasted exploration turns and makes every generated environment smarter from turn zero.

**Status:** Planned  
**Depends on:** v1.12.0  
**Inspired by:** [Meta-Harness](https://yoonholee.com/meta-harness/) (Stanford IRIS Lab / Chelsea Finn group, March 2026) — discovered via automated harness evolution on TerminalBench-2

---

## The Problem

Every agent session starts the same way: the agent runs `ls`, `pwd`, `which python3`, `node --version`, `cat package.json`, etc. to figure out what it's working with. This wastes 2-5 turns of context window and API cost on pure discovery — information the harness could have provided upfront.

Meta-Harness proved this empirically. Their optimizer independently discovered "Environment Bootstrapping" as the #1 improvement for agentic coding tasks. Before the agent loop starts, a single compound shell command gathers a snapshot of the sandbox (working directory, file listing, available languages/tools, package managers, memory) and injects it into the initial prompt. This alone moved their Haiku 4.5 agent to #1 on TerminalBench-2.

Kairn already generates rich CLAUDE.md files with project context. But those describe the *intended* workflow — not the *actual runtime state*. A bootstrapped environment snapshot closes this gap.

---

## The Fix: Two Complementary Approaches

Kairn generates environments for Claude Code, not for a custom agent runtime. So we can't run arbitrary shell commands before the agent loop. Instead, we inject bootstrapping behavior into the generated harness itself.

### Approach 1: Bootstrap Command (`/project:bootstrap`)

A generated slash command that agents can (and are instructed to) run first:

```markdown
# /project:bootstrap — Environment Snapshot

Run this command at the start of any session to gather runtime context.
This saves 2-5 exploratory turns.

1. Run the following compound command and read the output:
   ```bash
   echo '=== WORKING DIRECTORY ===' && pwd && \
   echo '=== PROJECT FILES ===' && ls -la && \
   echo '=== GIT STATUS ===' && git status --short 2>/dev/null || echo 'not a git repo' && \
   echo '=== LANGUAGES ===' && \
   (python3 --version 2>&1 || echo 'python3: not found') && \
   (node --version 2>&1 || echo 'node: not found') && \
   (go version 2>&1 || echo 'go: not found') && \
   (rustc --version 2>&1 || echo 'rustc: not found') && \
   (java -version 2>&1 | head -1 || echo 'java: not found') && \
   echo '=== PACKAGE MANAGERS ===' && \
   (npm --version 2>&1 && echo "npm $(npm --version)" || echo 'npm: not found') && \
   (pip3 --version 2>&1 || echo 'pip3: not found') && \
   (cargo --version 2>&1 || echo 'cargo: not found') && \
   echo '=== ENVIRONMENT ===' && \
   (cat .env 2>/dev/null | sed 's/=.*/=***/' || echo 'no .env file') && \
   echo '=== MEMORY ===' && free -h 2>/dev/null | head -2 || echo 'memory info unavailable'
   ```

2. Summarize the environment in 3-4 lines:
   - Runtime: [languages + versions]
   - Project: [framework, key deps, file count]
   - State: [git branch, clean/dirty, .env present]

3. Keep this summary in context for the rest of the session.
```

### Approach 2: SessionStart Hook with Bootstrap

Use the `settings.json` SessionStart hook to automatically print the snapshot when Claude Code starts:

```json
{
  "hooks": {
    "SessionStart": [
      {
        "matcher": "",
        "command": "echo '--- Environment Snapshot ---' && pwd && ls -la --color=never | head -20 && echo '---' && git status --short 2>/dev/null && echo '---' && (node --version 2>&1 || true) && (python3 --version 2>&1 || true)"
      }
    ]
  }
}
```

This runs automatically — the agent sees the output as its first context without needing to call any tool.

### Approach 3: CLAUDE.md Bootstrap Section

A static section in CLAUDE.md that instructs the agent to self-bootstrap:

```markdown
## First Turn Protocol

At the start of every session, before doing ANY work:
1. Run `pwd && ls -la && git status --short` to orient yourself
2. Check available runtimes: `node --version`, `python3 --version` (as relevant)
3. Read any TODO.md, SPRINT.md, or task-tracking files
4. Summarize what you see in 2-3 lines, then proceed with the task

This eliminates wasted exploration turns. Never ask "what files are here?" — look first.
```

---

## Which Approach for Which Autonomy Level

| Level | Bootstrap Strategy | Rationale |
|-------|-------------------|-----------|
| Level 1 (Guided) | Approach 3 (CLAUDE.md section) | User is present; agent should explain what it sees |
| Level 2 (Assisted) | Approach 1 (`/project:bootstrap`) | User triggers it; agent summarizes |
| Level 3 (Autonomous) | Approach 2 (SessionStart hook) | Automatic; agent gets context without user action |
| Level 4 (Full Auto) | Approach 2 (SessionStart hook) | Must be automatic; no user to trigger anything |

All levels also get Approach 3 (the CLAUDE.md instruction) as a fallback.

## Implementation

### Changes to `src/compiler/prompts.ts`

Add a "First Turn Protocol" section to the HARNESS_PROMPT template. This becomes part of every generated CLAUDE.md:

```typescript
const BOOTSTRAP_SECTION = `
## First Turn Protocol

At the start of every session, before doing ANY work:
1. Run \`pwd && ls -la && git status --short\` to orient yourself
2. Check available runtimes relevant to this project: {{RUNTIME_CHECK_COMMANDS}}
3. Read any task-tracking files (TODO.md, SPRINT.md, docs/SPRINT.md)
4. Summarize what you see in 2-3 lines, then proceed

This saves 2-5 exploratory turns. Never ask "what files are here?" — look first.
`;
```

The `{{RUNTIME_CHECK_COMMANDS}}` placeholder is filled based on the project type detected in Pass 1:
- Node.js project → `node --version && npm ls --depth=0`
- Python project → `python3 --version && pip list --format=columns | head -10`
- Rust project → `rustc --version && cargo --version`
- Go project → `go version`
- Mixed → all relevant checks

### Changes to `src/compiler/compile.ts`

In Pass 2 (harness generation), inject the bootstrap section into the CLAUDE.md template:

```typescript
// After getting the skeleton from Pass 1
const runtimeChecks = inferRuntimeChecks(skeleton.tools, skeleton.projectType);
const bootstrapSection = BOOTSTRAP_SECTION.replace(
  '{{RUNTIME_CHECK_COMMANDS}}',
  runtimeChecks
);
```

### Changes to `src/compiler/settings.ts` (Pass 3)

For autonomy levels 3-4, generate a SessionStart hook:

```typescript
function generateBootstrapHook(projectType: string): Hook {
  const checks = [
    "echo '--- Environment Snapshot ---'",
    "pwd",
    "ls -la --color=never | head -20",
    "git status --short 2>/dev/null || true",
  ];

  if (projectType.includes('node') || projectType.includes('typescript')) {
    checks.push("node --version 2>&1 || true");
    checks.push("cat package.json | head -5 2>/dev/null || true");
  }
  if (projectType.includes('python')) {
    checks.push("python3 --version 2>&1 || true");
  }

  return {
    matcher: "",
    command: checks.join(" && "),
  };
}
```

### Changes to `src/compiler/commands.ts`

Generate the `/project:bootstrap` command for autonomy levels 1-2:

```typescript
function generateBootstrapCommand(projectType: string): Command {
  // ... build the compound command based on project type
  return {
    name: '/project:bootstrap',
    content: bootstrapCommandMarkdown,
  };
}
```
